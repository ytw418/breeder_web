# 기획자 문서 인덱스

작성일: 2026-02-11

## 목적
서비스 방향성 문서를 실행 가능한 계획으로 변환해, 바로 개발 티켓으로 쪼갤 수 있는 상태를 만든다.

## 운영 원칙 (단일 오너)
- 의사결정 오너는 기획자 1명으로 고정한다.
- 디자이너/마케터/개발자는 `기획자` 문서를 단일 기준으로 실행한다.
- 목표/우선순위/반려 기준 변경은 기획자가 즉시 문서 반영한다.

## 문서 목록
- `00_단일오너_운영원칙.md`: 기획자 단일 오너 체계, 역할, 운영 리듬
- `서비스_기획서.md`: 서비스 전체 방향, 우선순위, KPI, 12주 로드맵
- `01_분기전략_OKR.md`: 2026 Q1 목표와 핵심 결과
- `02_MVP_PRD_v1.md`: 당장 만드는 기능의 요구사항/수용 기준
- `03_스프린트백로그_2주.md`: 2026-02-12 ~ 2026-02-25 실행 백로그
- `04_운영정책_리스크.md`: 운영 원칙, 분쟁 대응, 리스크 관리
- `05_카테고리_사전_v1.md`: 상품/게시글 카테고리 표준
- `06_경매툴100_작업지시.md`: 초기 사용자 100명 달성용 팀별 실행 지시

- `07_통합검수_체크리스트.md`: 기획자 승인/반려용 통합 검수 기준

## 이번 사이클 우선순위
1. 경매 시스템 완성도 확보
2. 경매 도구/생성기 메시지로 초기 사용자 100명 확보
3. 생성 -> 첫 입찰 전환율 개선
4. 재방문/재생성 루프 확보

## 운영 방식
- 모든 기능은 `문제 -> 가설 -> 구현 -> KPI 측정` 순서로 진행
- PR 단위에서 수용 기준(AC) 충족 여부를 체크리스트로 검증
- 주간 리포트는 KPI 편차와 다음 주 액션만 요약
