# 통합 검수 체크리스트 (기획자 승인용)

작성일: 2026-02-11  
사용 시점: 매일 마감 전 점검, 릴리즈 승인 전 최종 점검

## 1) 기획 검수
- [ ] 목표/KPI가 "활성 사용자 100명"에 직접 연결되어 있다.
- [ ] 기능별 수용 기준(AC)이 명확하다.
- [ ] 이벤트 추적 정의(생성/공유/입찰/완료/재방문)가 문서화되어 있다.
- [ ] 2단계 락인/수익화 진입 조건이 명시되어 있다.

## 2) 디자인 검수
- [ ] 생성 -> 공유 -> 참여 -> 입찰 -> 종료 흐름이 끊기지 않는다.
- [ ] 모바일 우선 화면과 에러 상태가 설계되어 있다.
- [ ] 카피/레이블이 "경매 도구/생성기" 포지셔닝과 일치한다.
- [ ] 개발 인계 가능한 컴포넌트 규칙이 있다.

## 3) 마케팅 검수
- [ ] 채널별 실행안(일정/담당/예산/KPI)이 있다.
- [ ] 랜딩 메시지와 제품 경험이 일치한다.
- [ ] 퍼널 지표(유입->생성->입찰->재방문) 측정 방법이 있다.
- [ ] 2주 내 100명 달성 경로가 숫자로 검증 가능하다.

## 4) 개발 검수
- [ ] 경매 정합성(입찰/자동연장/종료 정산) 테스트가 통과했다.
- [ ] 핵심 이벤트 로깅이 실제 코드에 반영됐다.
- [ ] 오류/예외 처리와 사용자 안내 문구가 존재한다.
- [ ] 운영/신뢰 이슈(신고/관리자 확인)가 대응 가능하다.

## 5) 통합 일관성 검수
- [ ] 기획-디자인-마케팅-개발 문서 간 충돌이 없다.
- [ ] "바이럴 도구"와 "중장기 락인" 전략이 하나의 로드맵으로 연결된다.
- [ ] 기준을 충족하지 못하면 즉시 반려하고 수정 항목을 지정했다.
