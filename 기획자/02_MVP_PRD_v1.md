# MVP PRD v1

작성일: 2026-02-11  
버전: v1.0  
대상 릴리즈: 2026-02-26

## 1. 배경
현재 서비스는 거래/커뮤니티/채팅 기반이 이미 존재하지만, 카테고리 일관성, 검색 품질, 알림 신뢰도에서 사용자 마찰이 남아 있다. 이번 MVP는 신규 기능 확장보다 기존 핵심 루프의 전환 개선에 집중한다.

## 2. 목표
- 사용자가 원하는 상품/게시글을 더 빨리 찾게 한다.
- 거래로 이어지는 대화 시작률을 높인다.
- 알림을 믿고 다시 방문할 수 있게 만든다.

## 3. 범위
포함:
- 카테고리 체계 정비
- 통합 검색 품질 개선
- 알림 정확도/가시성 개선

제외:
- 결제/에스크로
- 추천 알고리즘
- 신규 커뮤니티 대형 기능(도감/랭킹 대개편)

## 4. 기능 요구사항

### F1. 카테고리 체계 정비
요구사항:
- 상품 등록 시 카테고리 미선택 상태를 줄이기 위해 기본 선택 UX 제공
- 상품 목록/검색/상세에서 동일한 카테고리 명칭 사용
- `productType`, `category`, `status` 조합 필터를 표준화

수용 기준(AC):
- AC1. 상품 등록 시 카테고리 값이 `null`로 저장되는 비율이 10% 미만
- AC2. 목록 API에서 카테고리 필터 시 결과 일관성(샘플 30건 검증) 100%
- AC3. 카테고리/상태 필터 조합 요청의 응답 실패율 0%

### F2. 통합 검색 품질 개선
요구사항:
- 기존 `q`, `type` 기반 검색에서 정확도 보강
- 검색 결과가 상품/게시글/유저로 명확히 구분된 UI 제공
- 무결과 시 대체 탐색(추천 카테고리/인기 키워드) 제공

수용 기준(AC):
- AC1. `q` 입력 후 1초 내 첫 결과 렌더링(일반 환경 기준)
- AC2. 상위 20개 결과에서 완전 무관 결과 비율 10% 미만(내부 샘플 기준)
- AC3. 무결과 화면에서 재탐색 클릭률 15% 이상

### F3. 알림 신뢰도 개선
요구사항:
- 알림 타입별 메시지 템플릿 점검(중복/오해 표현 제거)
- 읽음 처리 및 unreadCount 일관성 보장
- 알림 클릭 시 대상 페이지로 안정적 이동

수용 기준(AC):
- AC1. 중복 알림 제보 50% 이상 감소(릴리즈 전후 2주 비교)
- AC2. unreadCount와 실제 미읽음 갯수 불일치 0건
- AC3. 알림 클릭 후 이동 실패율 1% 미만

## 5. 사용자 스토리
- US1. 구매자로서, 카테고리/상태로 상품을 빠르게 좁혀보고 싶다.
- US2. 사용자로서, 검색 결과에서 상품/게시글/유저를 한 번에 비교하고 싶다.
- US3. 판매자로서, 알림을 통해 문의/반응을 놓치지 않고 즉시 대응하고 싶다.

## 6. 비기능 요구사항
- API 응답 형식은 기존 `{ success: boolean, ... }` 패턴 유지
- 기능 추가로 기존 경매/채팅 플로우에 회귀 버그가 없어야 함
- 모바일 기준 주요 화면(목록/검색/알림)은 1스크린 내 핵심 액션 노출

## 7. 의존성
- 백엔드: `pages/api/products`, `pages/api/search`, `pages/api/notifications`
- 프론트엔드: 상품 목록, 검색 페이지, 알림 페이지
- 데이터: 카테고리 값 정규화 정책(마이그레이션 여부 판단 필요)

## 8. 출시 기준
- AC 충족 + 주요 시나리오 E2E 통과
- 운영팀 공지 문안 및 변경사항 FAQ 준비
- 장애 대응 담당자/롤백 기준 지정
