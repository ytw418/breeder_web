## 1. 서비스 한 줄 정의
곤충 애호가를 시작점으로, 거래와 커뮤니티를 결합해 신뢰 기반 생물 생태계를 만드는 전문 플랫폼.

## 2. 문제 정의
- 정보와 거래가 네이버 카페/오픈채팅/중고 플랫폼에 분산되어 탐색 비용이 높음
- 생물 거래 특성상 신뢰(혈통/사육 이력/거래 매너) 정보가 부족함
- 입문자는 사육 지식 접근이 어렵고, 숙련자는 전문성을 보여줄 공간이 부족함

## 3. 타겟 사용자
- 1차: 곤충 브리더, 취미 사육자, 용품 판매자, 입문자
- 2차: 파충류/어류 등 특수 반려 카테고리 사용자
- 3차: 일반 반려동물(강아지/고양이) 브리딩/전문 커뮤니티 사용자

## 4. 핵심 가치 제안
- `Trust`: 프로필, 거래 후기, 활동 이력으로 신뢰도 판단 가능
- `Focus`: 곤충/생물 중심 카테고리와 탐색 경험 제공
- `Community`: 거래 이후에도 남는 정보/관계 중심 구조
- `Growth`: 커뮤니티 지식이 쌓일수록 검색 유입과 재방문이 증가

## 5. 제품 구조 (IA)
- 홈 피드: 추천 게시글/상품/인기 콘텐츠
- 마켓: 상품 탐색, 필터, 거래 상태, 찜
- 커뮤니티: 자유/질문/정보/자랑 게시판
- 채팅: 1:1 거래 커뮤니케이션
- 알림: 댓글/좋아요/채팅/거래 상태
- 마이: 프로필, 활동 내역, 후기, 관심 목록
- 관리자: 유저/게시글/상품/경매 운영

## 6. 현재 기준 기능 상태
이미 구현/운영 중인 핵심 기능:
- 인증 및 프로필 관리
- 상품 CRUD
- 찜/팔로우/알림
- 1:1 채팅
- 경매 등록/입찰/종료 처리
- 관리자 페이지

## 7. 우선순위 기능 (MVP 고도화)

### P0 (즉시)
- 카테고리 체계 정비: 곤충 세부 카테고리 + 필터/정렬 일관성
- 통합 검색: 상품/게시글/사용자 검색
- 커뮤니티 댓글 품질 개선: 문의-답변 흐름 강화
- 알림 정확도 개선: 채팅/댓글/좋아요/거래 이벤트 실시간성 점검

### P1 (단기)
- 거래 상태 플로우 고도화: 판매중/예약중/완료 UX 명확화
- 리뷰/평판 시스템 개선: 후기 신뢰도 표시
- 커뮤니티 메뉴 활성화: 질문/노하우/자랑 중심 콘텐츠 구조

### P2 (중기)
- 종 도감/사육 가이드(SEO): 입문자 유입 채널 확보
- 기네스북/랭킹: 참여형 콘텐츠로 재방문 유도
- 혈통/브리딩 이력 기록: 프리미엄 거래 기반 데이터 구축

## 8. 차별화 전략
- 거래 + 전문 지식 + 브리딩 이력의 결합
- 일반 중고 앱이 제공하지 못하는 `종 특화 맥락` 제공
- 콘텐츠(도감/노하우)와 거래를 순환시키는 성장 구조 설계

## 9. 수익 모델
- 단기: 광고 슬롯, 유료 상위 노출, 프로모션 배지
- 중기: 프리미엄 구독(광고 제거/고급 노출/인증 기능)
- 장기: 업체 스토어 입점, 안전거래 수수료, 이벤트/전시 연계

## 10. 핵심 KPI
- 활성: DAU/WAU, 신규 가입 전환율
- 거래: 상품 등록 수, 거래 완료율, 메시지 시작률
- 커뮤니티: 게시글/댓글 수, 1인당 주간 상호작용
- 신뢰: 후기 작성률, 신고율, 분쟁 해결 시간
- 수익: 광고 CTR, 유료 노출 구매율, 구독 전환율

## 11. 12주 실행 로드맵 (제안)
- 1~2주: 카테고리/검색/알림 데이터 정합성 정리
- 3~4주: 마켓 탐색 UX + 거래 상태 플로우 개선
- 5~6주: 커뮤니티 게시판 구조 개편 + 댓글 경험 개선
- 7~8주: 리뷰/평판 모델 고도화
- 9~10주: 종 도감 MVP 오픈
- 11~12주: 랭킹/기네스북 PoC 및 성과 측정

## 12. 운영 정책 초안
- 금지 거래 품목/표현 명확화
- 신고/제재 정책 공개
- 거래 분쟁 처리 SLA 정의
- 개인정보/동물복지 관련 정책 준수 기준 문서화
