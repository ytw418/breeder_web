브리더(Breeder) - 곤충/동물 전문 커뮤니티 플랫폼 사업 기획서

1. 서비스 개요

한 줄 요약: 곤충 애호가를 위한 SNS + 중고거래 + 전문 커뮤니티 플랫폼

비전: 곤충에서 시작해 파충류, 물고기, 강아지, 고양이까지 아우르는 종합 반려동물 전문 플랫폼

핵심 가치:

곤충/동물 애호가들이 모이고, 소통하고, 거래하는 원스톱 플랫폼

기존 네이버 카페/밴드에 흩어진 커뮤니티를 하나로 통합

전문성 있는 콘텐츠와 신뢰 기반의 거래 환경 제공

2. 타겟 유저

1차 타겟 (론칭): 곤충 애호가

곤충 브리더 (사육/번식 전문가)

곤충 취미 사육자

곤충 용품 판매자

곤충에 관심 있는 입문자

2차 타겟 (확장): 파충류 -> 물고기 -> 강아지/고양이 순으로 확장

4. 수익 모델

4-1. 단기 수익 (론칭 후 즉시)

광고: 배너 광고, 피드 사이 네이티브 광고

프리미엄 상품 노출: 상품 상위 노출, 하이라이트 표시 (유료)

거래 수수료: 안전거래 시 수수료 (3~5%)

4-2. 중기 수익 (유저 확보 후)

생물업체 협업: 업체 공식 스토어 입점, 광고 계약

프리미엄 구독: 혈통 증명서 무제한 발급, 매칭 우선 노출, 광고 제거

용품 제휴: 사료/사육 용품 제휴 판매 (커미션)

4-3. 장기 수익 (플랫폼 확장 후)

동물 병원/펫샵 연동: 지역 기반 업체 광고

라이브 커머스: 생물/용품 라이브 판매

이벤트/전시: 곤충 박람회, 경매 이벤트 주최

5. 카테고리 확장 로드맵

Phase 1: 곤충 (장수풍뎅이, 사슴벌레, 타란튤라, 전갈 등)
|
Phase 2: 파충류 (뱀, 도마뱀, 거북이 등)
|
Phase 3: 물고기 (열대어, 해수어, 수초 등)
|
Phase 4: 강아지/고양이 (브리딩 전문)
|
Phase 5: 종합 반려동물 플랫폼

6. 기술 스택 (현재 기반)

Frontend: Next.js 14, TypeScript, TailwindCSS, SWR

Backend: Next.js API Routes, Prisma ORM

DB: PostgreSQL (Supabase)

이미지: Cloudflare Images

배포: Vercel

실시간 (향후): Supabase Realtime 또는 Firebase

7. 개발 우선순위 (뭐부터 만들면 좋을지)

현재 이미 갖고 있는 것: 인증, 상품 CRUD, 게시글, 채팅, 프로필, 팔로우, 리뷰

Phase 1 - 기반 정비 + 커뮤니티 강화 (1~2주)

카테고리 시스템 구축 - 곤충 종류별 카테고리 (장수풍뎅이, 사슴벌레 등) + 상품/게시글에 카테고리 필터

피드 개선 - 사진 중심 피드 UI, 좋아요/댓글 UX 강화

검색 기능 - 통합 검색 (상품 + 게시글 + 유저)

알림 시스템 - 기본 알림 (좋아요, 댓글, 채팅)

Phase 2 - 거래 고도화 (2~3주)

상품 카테고리 필터 - 생물/용품 분류, 가격대 필터, 지역 필터

거래 상태 관리 - 판매중/예약중/판매완료 플로우

거래 리뷰 개선 - 거래 완료 후 상호 리뷰 플로우

Phase 3 - 킬러 콘텐츠 (3~4주)

종 도감 - 곤충 종류별 정보 위키 (SEO 효과 큼)

곤충 기네스북 - 크기/무게 랭킹 시스템

사육 일지 - 성장 기록 타임라인

Phase 4 - 수익화 (4~6주)

광고 시스템 - 배너, 네이티브 광고 슬롯

프리미엄 상품 노출 - 유료 상위 노출

업체 스토어 - 생물업체 공식 입점 페이지

8. 추천: 가장 먼저 만들 것

"카테고리 시스템 + 검색"을 가장 먼저 하는 것을 추천합니다.

이유:

현재 상품/게시글에 카테고리 구분이 없어서 "곤충 커뮤니티"라는 정체성이 약함

카테고리가 있어야 유저가 원하는 콘텐츠를 찾을 수 있음 (리텐션)

향후 파충류/물고기 등 확장의 기반이 됨

Product 모델에 이미 category 필드가 있어서 빠르게 구현 가능

구체적으로:

곤충 카테고리 체계 설계 (대분류: 장수풍뎅이/사슴벌레/거미류 등)

상품 등록 시 카테고리 선택 UI

메인 페이지에 카테고리 탭/필터

카테고리별 상품 목록 + 검색
