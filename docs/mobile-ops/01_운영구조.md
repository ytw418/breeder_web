# 운영 구조 (핸드폰 전용)

작성일: 2026-02-12

## 1) 인력 배치
- 역할 A: 데이터 수집 담당
 - Yahoo/Naver/Instagram에서 거래형 데이터 수집
 - 카톡 템플릿으로 제출
- 역할 B: 어드민 매니저 보조
 - 공지 초안 작성
 - 신고 분류 1차 처리
 - 상품 숨김/검토 후보 정리

## 2) 도구
- 필수: 카카오톡, 휴대폰 메모 앱
- 선택: 네이버 앱, 인스타 앱, 브라우저
- 금지: 코드 에디터, JSON 파일 직접 수정

## 3) 운영 채널
- 카톡 단톡방 1개: `브리디-모바일운영`
- 메시지 태그 규칙
 - `#수집` 데이터 수집 제출
 - `#신고` 신고 분류 제출
 - `#공지` 공지 초안 제출
 - `#완료` 당일 업무 종료 보고

## 4) 하루 업무 루틴 (총 60~90분)
1. 오전(20분)
 - A: 신규 매물 10개 수집
 - B: 전일 신고 10건 분류
2. 오후(20분)
 - A: 추가 매물 10개 수집
 - B: 공지 초안 1건 작성
3. 마감(10분)
 - A/B 모두 `#완료` 템플릿 제출

## 5) KPI (초기)
- 수집 유효건수: 1인 20건/일
- 신고 분류 처리: 1인 10건/일
- 공지 초안: 1건/일
- 무효 데이터율(출처 누락/중복): 10% 이하
