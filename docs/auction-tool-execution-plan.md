# 링크형 경매 도구 실행 플랜 (1~5)

작성일: 2026-02-11  
목표: 커뮤니티 대체가 아니라, 카페/밴드에서 바로 붙여 쓰는 경매 도구 MVP를 6주 안에 안정화한다.

## 1. 목표 및 KPI

- 핵심 목표
  - 링크 생성 후 즉시 참여 가능한 경매 경험 제공
  - 기존 카페 경매 대비 입력 실수/분쟁/운영 피로를 줄이는 것
- 6주 KPI
  - 생성 경매 수: 100건 이상
  - 참여 사용자 수: 1,000명 이상
  - 셀러 재사용률: 40% 이상
  - 신고율(신고 건/완료 경매): 3% 이하
- 핵심 퍼널
  - 경매 생성 -> 링크 공유 -> 첫 입찰 -> 경매 완료 -> 셀러 재등록

## 2. MVP 범위 고정

- 포함
  - 경매 생성: 시작가/입찰단위/종료시간/자동연장
  - 참여: 링크 접근 -> 로그인 -> 즉시 입찰
  - 신뢰: 신고/제재/입찰 이벤트 로그 확인
  - 알림: 1등, 밀림, 낙찰, 종료
- 제외
  - 채팅, 결제/에스크로, 커뮤니티 피드, 추천 알고리즘

## 3. 실행 일정

- 1주차: 룰/정책 확정, 이벤트 정의, 운영 시나리오 정리
- 2주차: 입찰 엔진 및 자동연장/검증 로직 고도화
- 3주차: 신고/제재, 관리자 처리 흐름 추가
- 4주차: 웹푸시/알림 안정화 및 템플릿화
- 5주차: 파일럿(셀러 10명), 이슈 수집/개선
- 6주차: 베타 확장(셀러 30명), KPI 기준 Go/No-go 판단

## 4. 기술 우선순위

1. 서버 정합성
   - 동시 입찰, 최소 입찰 규칙, 종료 정산, 중복 알림 방지
2. 진입 장벽 최소화
   - 로그인 후 즉시 참여, 모바일 입력 UX 최적화
3. 운영 가능성
   - 신고 접수/검토/제재 흐름, 관리자 로그 확인
4. 성장 장치
   - 링크 공유, 재참여 유도 알림

## 5. 운영 룰 (초기 고정)

- 입찰 취소 불가 (운영자 예외 처리만 허용)
- 동가 처리: 먼저 도달한 입찰 우선
- 마감 임박 자동연장: 종료 3분 이내 입찰 시 5분 연장
- 종료 후 상태/낙찰자 산정은 서버 정산 결과를 기준으로 고정
- 신고 누적 또는 중대 위반 시 계정 영구정지
- 분쟁 처리 1차 근거: 입찰 이벤트 로그 + 신고 접수 내용
